## 详细描述（约 500-1000 字）

```
中文框选朗读是一个面向普通网页和视频网站的中文 TTS 助手。

【核心功能】
1. 框选朗读 — 在任意网页上拖出一个矩形，自动提取并朗读其中的中文文本，可调整选区大小后再确认。
2. 视频字幕跟读 — 框选 YouTube / Bilibili / Netflix / 在线课程的字幕区域并锁定，扩展会持续监听该区域，每当字幕滚动到新一句即自动朗读，与视频同步推进。锁定后边框可拖动调整。
3. 10 种音色预设 — 内置 Microsoft 中文神经语音匹配（晓晓 / 云希 / 云扬 / 云健 / 晓伊 / 云枫 / 云皓 / 云夏 / 云泽 / 晓辰），覆盖男声、女声、童声、长者声等场景。Edge 浏览器自带这些在线神经语音，效果最佳。
4. 音量增强（0%-200%）— 100% 以下控制 TTS 音量；100%-200% 时通过 Web Audio GainNode 同步放大网页内 video / audio，解决"视频本身声音太小"的问题。
5. 仅朗读中文 — 自动过滤英文与无关符号，对混排页面更友好。

【快捷键】
• Alt+Shift+R 开启 / 关闭框选朗读
• Alt+Shift+V 开启 / 关闭视频字幕跟读
• Alt+Shift+S 停止朗读
• Enter 确认调整中的选区
• Esc 退出当前模式

【使用场景】
• 中文阅读辅助、无障碍朗读
• 边看视频边练听力（朗读跟随字幕滚动）
• 长文章一键听书

【隐私】
扩展不会上传你的浏览数据，全部朗读基于浏览器原生 Web Speech API 在本地完成。设置数据保存在 chrome.storage 中。可选的留言板功能需要用户自行配置 Firebase 后端，扩展本身不收集留言。

【技术】
基于 Manifest V3，Web Speech API 朗读，TreeWalker + getClientRects 提取文本，MutationObserver + setInterval 双轨监听字幕变化。
```

## 权限说明（Chrome Web Store / Edge 必填，逐项解释为什么需要）

| 权限 | 用途 |
|---|---|
| `activeTab` | 当用户点击扩展图标或快捷键时，向当前标签页注入框选朗读的 UI 与脚本。 |
| `storage` | 保存语速 / 音色 / 选框颜色等本地设置；保存可选的 Firebase 配置与登录会话。 |
| `contextMenus` | 在网页右键菜单提供"朗读选中文本"等快捷入口。 |
| `scripting` | 在用户主动开启时向当前页注入框选与字幕监听代码（按需，不在后台扫描所有页面）。 |
| `host_permissions: <all_urls>` | 由于用户可能在任意网站使用框选/字幕跟读，需要可访问任意页面。**仅在用户主动开启功能时才会读取页面文本，不会在后台扫描浏览数据**。 |

## 隐私政策摘要（粘贴到表单或链接到 publishing/PRIVACY.md）

```
本扩展不收集、不存储、不传输任何用户浏览数据或个人信息。

- 朗读功能完全运行在用户本地浏览器（Web Speech API），无任何云端 TTS 调用。
- 所有用户偏好（语速、音色、选框颜色等）保存在浏览器的 chrome.storage 中，不会同步到第三方。
- 留言板与登录注册功能为可选，且需要用户自行在自己的 Firebase 项目中配置；扩展开发者不持有任何后端，也无法访问任何用户数据。
- 扩展不会包含任何分析、追踪或广告 SDK。

如有疑问，请通过商店列表中的开发者邮箱联系。
```
